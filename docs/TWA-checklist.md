# Telegram Mini App — Чеклист требований и листинга

> Актуально на март 2026. Основано на официальной документации Telegram, Bot Platform Developer ToS, и практическом опыте прохождения модерации.

---

## 1. Базовые технические требования

- [x] Mini App — это веб-приложение (HTML/CSS/JS), открывается внутри Telegram WebView
- [x] Бот создан через @BotFather, URL приложения привязан через `/setmenubutton` или `/myapps`
- [x] Вызывается `Telegram.WebApp.ready()` как можно раньше при загрузке — до отображения интерфейса
- [x] Подключён `telegram-web-app.js` (через CDN или `@twa-dev/sdk` для TypeScript)
- [x] Приложение работает корректно на Android (Chromium 119 WebView), iOS (WKWebView) и Desktop
- [x] На iOS добавлен `<meta name="color-scheme" content="light dark">` — иначе будет "flash of unstyled content"

---

## 2. Дизайн и UX

### 2.1 Обязательные принципы

- [x] Mobile-first, отзывчивый дизайн — адаптация под разные размеры экранов
- [x] Интерактивные элементы повторяют стиль и поведение нативных UI-компонентов Telegram
- [ ] Анимации плавные — 60fps
- [ ] Все поля ввода и изображения содержат accessibility-лейблы _(partial: img alt есть, aria-label на input/button нет)_
- [x] Приложение подхватывает динамические цвета темы Telegram через API (Day/Night/Custom themes)
- [x] Интерфейс учитывает `safeAreaInset` и `contentSafeAreaInset` (особенно в fullscreen-режиме)

### 2.2 Нативные элементы Telegram

- [x] Используется `MainButton` (нижняя кнопка) вместо кастомных CTA
- [x] Используется `BackButton` для навигации назад
- [ ] При необходимости — `SettingsButton` в контекстном меню _(не используется — пока не нужен)_
- [x] Haptic feedback через `HapticFeedback` API для тактильной отдачи

### 2.3 Рекомендации

- [x] Быстрая загрузка — кэширование данных, минимум серверных запросов, оптимизация кода
- [x] Минимализм: избегать сложных структур и многоуровневых меню
- [x] Использовать палитру цветов Telegram (обновлённую с версии 6.10): `bg_color`, `secondary_bg_color`, `section_bg_color`, header colors
- [ ] Figma UI Kit: https://www.figma.com/community/file/1348989725141777736

## 3. Платежи

### 3.1 Цифровые товары — ТОЛЬКО Telegram Stars

- [x] Подписки, внутриигровые валюты, доп. уровни, расширенное время игры, виртуальные предметы — оплата **исключительно** через Telegram Stars _(N/A — нет цифровых товаров)_
- [x] НЕ использовать сторонние платёжные системы для цифровых товаров — Telegram заблокирует приложение _(N/A — нет цифровых товаров)_
- [x] Определения "цифровых товаров" соответствуют политикам Apple (3.1.1) и Google (Payment Policies 1, 2, 4) _(N/A)_

### 3.2 Физические товары и услуги

- [x] Для физических товаров можно использовать любых сторонних платёжных провайдеров _(N/A — нет физических товаров)_
- [x] Поддержка Google Pay и Apple Pay через Bot Payments API _(N/A)_

### 3.3 Обязательные платёжные требования

- [ ] Бот отвечает на команду `/paysupport` — обработка вопросов пользователей по платежам _(N/A пока нет платежей, но нужно для листинга если будут Stars)_
- [x] Реализована возможность возврата Stars через `refundStarPayment` метод Bot API _(N/A — нет Stars)_
- [x] Цифровые товары доставляются вовремя и как описано _(N/A — нет цифровых товаров)_

### 3.4 Экономика Stars

- Разработчик получает ~$0.013 за 1 Star
- Stars можно потратить на рекламу в Telegram Ads ($0.02 за Star) — реинвестирование в продвижение
- Stars можно вывести через Fragment (конвертация в TON)

## 6. Листинг на Telegram Apps Center (@tapps_bot)

### 6.1 Подготовка

- [ ] Приложение **полностью готово** — "почти готовые" приложения отклоняют
- [x] Бот отвечает на `/start` **на английском** по умолчанию
- [ ] Подключён **@DataChief_bot** (Telegram Mini Apps SDK аналитика) — должен показывать свежие записи
- [x] Есть **Terms of Use** и **Privacy Policy** _(`/terms`, `/privacy` — Worker routes)_
- [x] Если крипто: только TON + TON Connect _(крипто пока нет, но план — TON only)_
- [x] Web3-интеграция **НЕ обязательна** для листинга — принимаются и некрипто-приложения

### 6.2 Подача заявки (через @app_moderation_bot)

- [ ] **Название** — короткое, запоминающееся
- [ ] **Тэглайн** — одна чёткая строка, чем приложение особенно
- [ ] **Скриншоты** (6 слотов) — оформлены как рекламные баннеры, а не просто скриншоты UI
- [ ] **Описание** — живым, человеческим языком; без корпоративного буллшита
- [ ] **Категория** — выбрана корректно

### 6.3 Модерация

- [ ] Ожидание: **3–8 дней**
- [ ] Могут дать фидбэк с просьбой исправить
- [ ] Баг-фикс билды можно пушить, пока заявка в ревью — новый билд заменяет позицию в очереди
- [ ] При отказе — ответить в тред с скриншотом фикса, обычно перепроверяют за 12 часов

### 6.4 Что даёт листинг

- Доступ к 10M+ пользователям платформы
- Органический приток 10K–1M+ пользователей
- Exposure через Trending Apps (канал с 6M+ подписчиков)
- Возможность Featured-размещения (для приложений с платежами в Stars и качественным контентом)
